## 📌 Prompt Text (v1.1 — current)
```
You are a procurement manager at an e-commerce fulfilment company. Draft a formal Request for Quote (RFQ) email to [SUPPLIER_NAME] for [PRODUCT_NAME]. We require [QUANTITY] units with delivery to [WAREHOUSE_ADDRESS] by [REQUIRED_DATE]. Request the supplier to provide: (1) unit price at requested quantity, (2) available quantity if stock is limited, (3) lead time from order confirmation, (4) minimum order quantity (MOQ), (5) payment terms offered. Quote reference: [RFQ_NUMBER]. Deadline for response: [QUOTE_DEADLINE]. Max 220 words. Format as a structured table request. Tone: professional and non-committal — this is a quote request only, not a commitment to purchase.
```

**Placeholders to fill:**

| Placeholder | Source | Example |
|-------------|--------|---------|
| `[SUPPLIER_NAME]` | Supplier database | AusPack Supplies Pty Ltd |
| `[PRODUCT_NAME]` | Product catalogue | Cardboard Shipping Boxes Medium |
| `[QUANTITY]` | Procurement plan | 500 units |
| `[WAREHOUSE_ADDRESS]` | Company records | Unit 4, 28 Logistics Drive, Derrimut VIC 3026 |
| `[REQUIRED_DATE]` | Procurement calendar | 30 November 2024 |
| `[RFQ_NUMBER]` | Procurement system | RFQ-2024-0094 |
| `[QUOTE_DEADLINE]` | Procurement calendar | 18 November 2024 |

## ❗ Problem Being Solved

Ad-hoc RFQ emails are inconsistently structured, often missing key data points (MOQ, lead time, payment terms), resulting in incomparable quotes requiring multiple follow-up rounds. Running 5 RFQs previously took 2+ hours. With this prompt, total time reduces to under 15 minutes including review — an 85% reduction enabling faster procurement decisions and better supplier comparison.

**Pain points addressed:**
- Inconsistent RFQ structure returns incomparable quotes — adds days to procurement cycle
- Missing data points (MOQ, payment terms) require costly follow-up rounds
- 2+ hours to draft and send 5 RFQs manually — delays stock availability decisions
- Commitment language in previous emails created legal ambiguity

## ⚠️ Risks and Limitations

| Risk | Level | Mitigation |
|------|-------|------------|
| Non-committal tone instruction ignored if AI infers purchase intent | High | Test output specifically for commitment language before use — added to prompt testing checklist |
| Quote deadline unrealistic for international suppliers | Medium | Set deadline variable based on supplier location — domestic 5 days, international 10 days |
| Sensitive pricing data in RFQ responses | Medium | Restrict RFQ access to procurement staff — quote prices stored in secure procurement system only |
| Supplier uses RFQ as contract evidence | Low | Include explicit non-commitment statement in prompt output — reviewed by manager before send |

**Overall risk rating:** LOW-MEDIUM — non-committal tone constraint and human review gate manage primary legal risk.

## 🔄 Version History

### v1.0 — Initial draft
**Date:** 31 March 2026  
**Prompt:** You are a procurement manager. Write an email to a supplier asking for a quote on a product we need.  
**Output:** Short email sounding like a purchase order — implied commitment to buy. No specific data points, no deadline, no RFQ reference. Legally risky.  
**Observed effect:** Required complete rewrite — legally ambiguous output.  
**Lesson learned:** RFQ emails must explicitly state non-commitment, request 5 specific data points in table format, and include a quote deadline and reference number.

---

### v1.1 — Added 5-point data request + non-committal tone + table format + deadline ✅ Current
**Date:** 31 March 2026  
**Change:** Added procurement manager RACE role, 7 placeholder variables, explicit 5-point structured data request, non-committal tone instruction, table format requirement, RFQ reference number, quote deadline, and 220-word limit.  
**Output:** Professional, legally sound RFQ email. Suppliers return comparable structured quotes. No commitment implied. No editing required.  
**Observed effect:** Time to send 5 RFQs reduced from 2+ hours to under 15 minutes — 85% saving. Comparable structured responses eliminate follow-up rounds.  
**Lesson learned:** Non-committal tone must be explicitly stated in the prompt — AI infers purchase intent from context without this instruction. Table format request drives comparable supplier responses essential for efficient comparison.
