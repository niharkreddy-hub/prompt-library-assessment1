## 📌 Prompt Text (v1.1 — current)
```
You are a supply chain analyst. Generate a weekly supplier performance summary report for the period [WEEK_START] to [WEEK_END] based on the following data: Supplier Name: [SUPPLIER_NAME] | On-Time Delivery Rate: [OTD_RATE]% | POs Issued: [PO_COUNT] | POs Confirmed Within 48hrs: [CONFIRMED_COUNT] | Discrepancies Reported: [DISCREPANCY_COUNT] | Discrepancy Value: $[DISCREPANCY_VALUE] AUD. Structure the report as: (1) Performance Summary (2 sentences), (2) Key Issues (bullet points, max 3), (3) Recommended Actions (bullet points, max 3), (4) Risk Rating: Low / Medium / High with one-sentence justification. Max 200 words. Use business language suitable for a procurement manager.
```

## ❗ Problem Being Solved

Procurement managers spend 2–3 hours every Monday manually compiling supplier performance data from multiple systems. Inconsistent reporting format makes week-to-week comparison impossible and at-risk suppliers are only identified after disruption has occurred. At $45/hr fully-loaded cost, Monday reporting alone costs $90–$135 per week — $4,680–$7,020 annually.

**Pain points addressed:**
- 2–3 hours weekly manual reporting — $4,680–$7,020 annual cost
- Inconsistent format prevents week-to-week performance trending
- At-risk suppliers identified reactively rather than proactively

---

## ⚠️ Risks and Limitations

| Risk | Level | Mitigation |
|------|-------|------------|
| Misleading risk ratings if input data contains errors | High | Validate all WMS data against source before prompt execution |
| Recommended actions lack supplier relationship context | Medium | Procurement manager reviews all actions before implementation |
| Reports used in contract disputes without review | Medium | Label all outputs as internal drafts — require manager sign-off |
| Consistent format may mask nuanced supplier dynamics | Low | Manager adds qualitative notes for complex relationships |

**Overall risk rating:** LOW-MEDIUM — structured data inputs eliminate hallucination risk. Human review gate prevents misuse.
